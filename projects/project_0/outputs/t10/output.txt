### Deployment Steps

#### Step 1: Prepare the Environment
```bash
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install Docker and Docker Compose
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# Verify installation
docker --version
docker-compose --version
```

#### Step 2: Transfer Build Artifacts
```bash
# Create deployment directory
mkdir -p /opt/myapp/deployment
cd /opt/myapp/deployment

# Copy build artifacts (replace with actual artifact paths)
scp -r user@build-server:/path/to/build_artifacts/* .
```

#### Step 3: Configure Environment Variables
Create a `.env` file in the deployment directory:

```bash
cat > .env << EOF
APP_ENV=production
NODE_ENV=production
PORT=3000
DB_HOST=db
DB_PORT=5432
DB_NAME=myapp_db
DB_USER=postgres
DB_PASSWORD=${POSTGRES_PASSWORD}
JWT_SECRET=${JWT_SECRET}
LOG_LEVEL=info
EOF

# Generate secure secrets (run once)
export POSTGRES_PASSWORD=$(openssl rand -base64 32)
export JWT_SECRET=$(openssl rand -base64 32)
```

#### Step 4: Deploy with Docker Compose
```bash
# Start services
docker-compose up -d --build

# Verify deployment
docker ps
docker logs myapp-app-1
```

#### Step 5: Configure Reverse Proxy (Optional but Recommended)
For production, use Nginx as a reverse proxy:

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### Step 6: Verify Deployment
```bash
# Test application endpoint
curl -s http://localhost/health | jq .

# Check logs for errors
docker-compose logs --tail=50 app

# Run database migrations (if applicable)
docker exec myapp-app-1 npm run migrate:up
```
