# Reference Documentation

These files contain detailed reference material for the Vetinari project. They are NOT loaded automatically — read them when you need specific guidance.

| File | Content | When to Read |
|------|---------|-------------|
| `architecture.md` | System architecture, components, key files, agent roles | When working on system design or unfamiliar with the codebase |
| `code-patterns.md` | Preferred patterns for I/O, exceptions, config, agents, tests | When implementing new features or unsure of the right pattern |
| `common-errors.md` | Error recovery guide for import, test, build, and git failures | When you encounter an error and need to debug |
| `import-patterns.md` | Import conventions, anti-patterns, adding new agent types | When writing imports or adding new agents |
| `logging-guide.md` | Logging setup, levels, formatting, print-to-logging migration | When adding logging or migrating from print() |
| `workflow.md` | Branch naming, commit conventions, PR requirements | When committing code or creating PRs |
| `writing-guide.md` | Writing standards for docstrings, comments, markdown, changelogs | When writing documentation or reviewing doc quality |
