# Research Methods

## Research Process

### 1. Define Scope
- Clarify the question
- Set boundaries
- Identify key terms

### 2. Gather Sources
- Primary sources first
- Multiple perspectives
- Recent and historical

### 3. Verify Information
- Cross-reference
- Check dates
- Validate claims
- Note confidence

### 4. Analyze
- Synthesize findings
- Identify patterns
- Note contradictions

### 5. Report
- Structure clearly
- Cite sources
- Acknowledge gaps

## Source Types

### High Credibility
- Official documentation
- Peer-reviewed papers
- Official statistics
- Expert consensus

### Medium Credibility
- Industry reports
- Technical blogs
- Conference talks
- Case studies

### Lower Credibility
- Social media
- Unverified forums
- Outdated docs
- Vendor marketing

## Verification Checklist

- [ ] Source exists and is accessible
- [ ] Author/organization is credible
- [ ] Date is recent (check for updates)
- [ ] Claims are supported by evidence
- [ ] Multiple sources agree
- [ ] Any conflicts are noted

## Search Strategies

### For Technical Topics
1. Official docs
2. GitHub issues/discussions
3. Stack Overflow (accepted answers)
4. Recent blog posts

### For Best Practices
1. Industry standards
2. Official guides
3. Expert recommendations
4. Case studies

### For Comparisons
1. Feature matrix from official sources
2. Performance benchmarks
3. Community feedback
4. Real-world usage reports

## Documentation Format

### Source Citation
```markdown
[1] Title
    Source: [URL]
    Date: [YYYY-MM-DD]
    Credibility: [High/Medium/Low]
```

### Finding Format
```markdown
**Finding**: [Key point]
**Confidence**: [High/Medium/Low]
**Source**: [1], [2]
**Notes**: [Any caveats]
```
